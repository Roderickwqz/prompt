# 01 Plan: Refactor + Optimize `src/alpha/report/factor.py`

## Objective
Refactor report module into presentation-only layer; optimize report runtime/memory; keep legacy report APIs stable.

Target outcomes:
- full funnel-stage report coverage from `src/alpha/pipeline/analysis/factor.py`
- single canonical funnel report contract (tables first, figures derived)
- zero duplicated business logic in report layer
- explicit stage status for `ok/skipped_disabled/unimplemented/error`
- measurable performance improvement with regression guardrails

## Source of Truth (Sync First)
Must read before edits:
- `docs/project/context.md`
- `docs/project/design.md`
- `docs/project/registry.md`
- `docs/principle/engineer.md`
- `docs/principle/code.md`

Registry maintenance:
- canonical registry file: `docs/project/registry.md`
- regenerate by: `python docs/scripts/registry.py --path ./src > docs/project/registry.md`

## Scope
In:
- `src/alpha/report/factor.py`
- `src/alpha/report/__init__.py`
- `tests/test_factor_report.py`
- `docs/project/registry.md`
- `docs/project/context.md` (only if architecture description changes)
- `docs/project/design.md` (only if boundary/rules change)
- this plan

Out:
- analyzer business logic in `src/alpha/pipeline/analysis/factor.py`
- `core/formulas/*` and `core/metrics/*` algorithm changes
- notebooks
- model report

## Design Constraints
- report layer = orchestration + normalization + plotting only
- no direct business-stat recomputation if analyzer output exists
- dependency direction:
  - allow `report -> pipeline.analysis.factor`
  - allow `report -> pipeline.analysis.univariate`
  - avoid direct `report -> core.formulas.*` business compute
- keep blast radius local to report module
- fail fast on invalid API inputs
- graceful degradation for empty/missing/unimplemented stages
- structured logging for stage failures (no `print`)
- public APIs keep type hints + docstrings

## Legacy Compatibility Rule
Keep these APIs callable and behavior-compatible (except bug fixes):
- `plot_factor_analysis_overview`
- `plot_factor_ic_decay`
- `plot_factor_batch_summary`
- `generate_factor_report_figures`

If temporary exception needed for legacy turnover (`compute_long_short_turnover`), declare it explicitly; do not add new direct business-compute dependencies.

## Canonical Funnel Contract
Primary APIs:
- `generate_factor_funnel_report_tables(ctx, *, respect_config=True) -> dict[str, pd.DataFrame]`
- `generate_factor_funnel_report_figures(ctx, *, tables=None) -> dict[str, Figure]`
- `generate_factor_funnel_report(ctx, *, respect_config=True) -> dict[str, dict]`

Required tables:
- `stage_status`
- `card_summary`
- `screening`
- `orthogonal`
- `redundancy`
- `multiplicity`
- `selection_bias`
- `bootstrap_null`
- `threshold`
- `oos`
- `capacity`
- `incremental`
- `bayesian`
- `review`
- `tiers`

Stage status schema:
- index: `stage`
- cols: `status`, `n_rows`, `error`
- status enum: `ok`, `skipped_disabled`, `unimplemented`, `error`

## Optimization Targets (Mandatory)
Dataset for benchmark: fixed deterministic fixture in tests.

KPIs:
- wall-time for `generate_factor_funnel_report` improved >= 20% vs baseline OR <= agreed absolute threshold
- peak memory not worse than baseline by > 10%
- no regression in output keys/schema/content contract

Method:
- capture baseline before refactor
- capture after refactor
- store benchmark note in plan PR description / test evidence

## Phases and Atomic Steps

### Phase 1: Baseline + Contracts
1. Snapshot current public APIs and output keys.
2. Add/confirm contract tests for keys, schemas, legacy API stability.
3. Record perf baseline (runtime + memory) on fixed fixture.

Exit:
- baseline numbers recorded
- contract tests red/green ready

### Phase 2: Control-Flow Unification
1. Define single stage spec registry (stage, analyzer, config gate).
2. Route all funnel stages through one runner path.
3. Normalize stage result handling and error capture.
4. Add structured logs for stage exceptions.

Exit:
- zero per-stage ad hoc execution logic
- `stage_status` always emitted

### Phase 3: Duplication Removal + Boundary Cleanup
1. Audit direct compute paths in report layer.
2. Replace with analyzer/context consumption where available.
3. Keep only presentation-local helpers.
4. Document any temporary legacy exception explicitly.

Exit:
- no new business-compute duplication in report
- dependency boundaries preserved

### Phase 4: Tables-First Architecture
1. Ensure each reportable stage has normalized table slot.
2. Build `card_summary` from stable card fields.
3. Make figures read tables first; only use `ctx.batch` for artifacts not yet tabularized (e.g., corr matrix).

Exit:
- figures are downstream of normalized tables
- notebook/test composition stable

### Phase 5: Figure Surface Rationalization
1. Keep minimal high-signal figures: stage status, screening scatter, confidence ranking, tier distribution, redundancy heatmap.
2. Standardize empty-data rendering behavior.
3. Ensure figure bundle keys stable.

Exit:
- small, coherent figure API
- no special-case explosion

### Phase 6: Robustness + Code-Standard Compliance
1. Add fail-fast validation for required inputs/columns/config assumptions.
2. Ensure public functions have full type hints + docstrings.
3. Replace any `print`-style diagnostics with structured logging.
4. Minimize unnecessary DataFrame copies where safe.

Exit:
- `code.md` robustness/doc/readability requirements met

### Phase 7: Validation + Docs Sync
1. Run targeted tests (`tests/test_factor_report.py`) and relevant suite.
2. Verify happy path + edge cases: empty tables, disabled stages, unimplemented stages, analyzer exceptions.
3. Regenerate/update `docs/project/registry.md` if public APIs changed.
4. Update `docs/project/context.md` and `docs/project/design.md` only if architectural guidance changed.

Exit:
- tests pass
- docs consistent with shipped APIs

## Acceptance Criteria
- covers all funnel analyzers with explicit report visibility
- canonical tables + figures contracts stable and tested
- legacy single-factor surfaces remain supported
- report layer does not duplicate quant business logic
- explicit status handling for disabled/unimplemented/error
- measurable perf target met (or documented fail with rationale + follow-up)
- docs and registry synced

## Risks and Controls
- risk: over-refactor breaks legacy API
  - control: contract tests before edits
- risk: optimization harms readability
  - control: KISS + small helpers + docstrings
- risk: hidden dependency creep
  - control: dependency audit in Phase 3
- risk: flaky perf checks
  - control: deterministic fixture + repeated runs

## Definition of Done
- plan executed phase-by-phase
- tests green
- perf evidence attached
- docs synced
- only scoped files changed
